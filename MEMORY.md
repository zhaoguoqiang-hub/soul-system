## 正负反馈双轨记录（2026-04-03 新增）

> 从 Claude Code 源码学到的：不仅要记错误修正，还要记成功验证。

### 记录时机
- 强哥明确说"效果好"、"这个思路不错" → 记正向
- 强哥纠正方向、说"不对"、"不是这个意思" → 记负向
- 任务完成后主动问一句"这次效果怎么样？"

### 记录格式
| 反馈类型 | 内容摘要 | 来源/任务 | 时间 |
|----------|----------|-----------|------|
| 正向 | 某次创作方向命中用户意图 | JJC-xxx | YYYY-MM-DD |
| 负向 | 调研方向偏差，后调整 | JJC-xxx | YYYY-MM-DD |

### 使用原则
- 正向反馈积累 → 未来主动推荐类似方案
- 负向反馈积累 → 未来避免类似错误
- 每月审查一次正负比例，负向过多说明分拣有问题

## Session Memory 机制（2026-04-03 新增）

> 来自 Claude Code 三层记忆架构的启发。

### 触发条件
在以下情况时，主动写一条 session summary 到 narrative-memory：
- 对话自然停顿（比如强哥去开会了）
- 任务完成时
- 每完成一个复杂阶段（调研完成/创作完成/审核完成）
- HEARTBEAT 检测到会话过长时

### 写入内容
```
[Session Summary] YYYY-MM-DD HH:MM
- 主题：
- 关键决策：
- 待跟进：
- 强哥情绪/能量状态（如果明显）：
```

### 整合规则（DreamTask 前置）
- 每周日凌晨 3 点（cron）检查 narrative-memory
- 如果积累 ≥5 条 session summary 且距上次整合 ≥24h
- 触发 DreamTask 四阶段：Orient → Gather → Consolidate → Prune
