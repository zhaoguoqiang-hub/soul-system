# 灵魂系统弹性机制

## 📋 定时任务清单

| 任务 | 频率 | 用途 | 风险 | 兜底机制 |
|------|------|------|------|----------|
| value-guard-boundary | 每30分钟 | 边界检查 | 错过几次影响小 | 每次执行独立检查 |
| value-guard-full | 每天3:00 | 全面评估 | 价值数据滞后 | 状态文件记录上次运行 |
| narrative-daily | 每天2:00 | 每日分析 | 叙事记忆丢失 | 检查daily_context.json |
| signal-distributor | 每15分钟 | 信号分发 | 信号堆积 | 系统启动时清理 |
| system-watchdog | 每小时 | 健康检查 | 系统状态未知 | 多级检查+补偿 |

## 🛡️ 兜底机制设计

### 1. 系统守护进程 (`system-watchdog.js`)
**功能：**
- ✅ 任务状态监控（检查最后执行时间）
- ✅ 错失任务补偿（自动执行漏掉的任务）
- ✅ 系统启动恢复（重启后执行关键任务）
- ✅ 健康状态检查（信号、上下文、任务）

**调度：**
- 每小时：健康检查 (`0 * * * *`)
- 每天1:00：深度恢复检查 (`1 1 * * *`)
- 系统启动：启动恢复 (`@reboot`)

### 2. 信号系统弹性
**问题：** 信号堆积在pending.jsonl中
**解决方案：**
- 信号分发器检查超时信号（>24小时）
- 系统启动时自动运行分发器
- 守护进程监控信号数量

### 3. 数据完整性保护
**每日上下文：**
- 检查日期是否为今天
- 检测未处理的关键事件
- 触发narrative-daily进行补偿处理

**叙事记忆：**
- 检查narrative.jsonl文件存在性
- 确保每日分析已执行

### 4. 任务漏执行补偿
**补偿逻辑：**
1. 检查日志文件的最后修改时间
2. 对比当前时间和任务调度频率
3. 如果超过最大允许错过时间，执行补偿
4. 记录补偿结果到watchdog-state.json

**最大错过时间配置：**
- 边界检查：2小时
- 信号分发：60分钟
- 每日任务：1-2天

## 🚀 系统启动恢复流程

```
系统启动
  ↓
运行 system-watchdog.js startup
  ↓
1. 强制执行所有关键任务
   - narrative-daily
   - signal-distributor
  ↓
2. 检查信号堆积
   - 如果>10个信号，立即运行分发器
  ↓
3. 检查每日上下文
   - 如果过期或有未处理事件，运行narrative-daily
  ↓
4. 执行健康检查
   - 检查所有任务状态
   - 执行必要的补偿
```

## 🔧 关键配置参数

### 任务配置 (`system-watchdog.js`)
```javascript
maxMissedHours: 2,      // 边界检查最多错过2小时
maxMissedMinutes: 60,   // 信号分发最多错过60分钟
maxMissedDays: 1,       // 每日任务最多错过1天
critical: true/false    // 是否为关键任务
```

### 信号堆积阈值
- 警告：>10个待处理信号
- 严重：>50个待处理信号
- 立即处理：系统启动时

### 补偿记录
- 保留最近100条补偿记录
- 记录补偿原因、时间、结果
- 存储在watchdog-state.json中

## 📊 健康检查项目

1. **任务执行状态** - 检查每个cron任务最后运行时间
2. **信号系统** - 检查pending.jsonl信号数量
3. **每日上下文** - 检查日期和未处理事件
4. **叙事记忆** - 检查narrative.jsonl文件
5. **系统状态** - 检查watchdog-state.json完整性

## 🚨 故障恢复场景

### 场景1：系统关机24小时后重启
1. 启动恢复检测到系统长时间离线
2. 强制执行所有关键任务
3. 检查并处理堆积的信号
4. 更新过期的每日上下文
5. 恢复正常调度

### 场景2：cron服务故障
1. 健康检查发现任务漏执行
2. 执行补偿任务
3. 记录故障信息
4. 继续监控直到cron恢复

### 场景3：磁盘空间不足
1. 检查日志文件大小
2. 清理旧日志文件
3. 确保关键文件可写
4. 记录存储警告

## 🎯 设计原则

1. **独立恢复** - 每个组件都能独立恢复，不依赖其他组件
2. **状态感知** - 系统知道自己的状态和上次运行时间
3. **渐进补偿** - 先补偿关键任务，再处理次要任务
4. **错误容忍** - 单个任务失败不影响整个系统
5. **状态持久化** - 所有状态变化都记录到文件

## 🔄 持续改进

1. **监控反馈** - 分析补偿记录，优化调度
2. **阈值调整** - 根据运行情况调整错过时间阈值
3. **新增检查** - 根据需要添加新的健康检查项
4. **性能优化** - 减少健康检查的资源消耗
