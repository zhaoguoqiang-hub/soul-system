# Feedback Tracker Skill

## 触发条件

**主动扫描 + 关键词触发**：
- 强哥说"效果好"、"不错"、"命中了"、"对了" → 正向反馈
- 强哥说"不对"、"不是"、"偏了"、"重来" → 负向反馈
- 任务完成后主动问"这次效果怎么样？"

## 记录格式

写入 `memory/feedback-log.json`，每条记录：
```json
{
  "id": "fb-{timestamp}",
  "timestamp": "2026-04-03T12:00:00+08:00",
  "type": "positive|negative",
  "taskId": "JJC-20260403-xxx 或 null",
  "summary": "一句话反馈摘要",
  "details": "详细描述（什么任务、哪里做得好/哪里出了问题）",
  "autoExtracted": true
}
```

## 自动提取规则

### 正向反馈关键词
- "效果好"、"不错"、"命中"、"对了"、"就是这感觉"
- "这个思路可以"、"有进步"
- 语气明显积极（"牛逼"、"绝了"）

### 负向反馈关键词
- "不对"、"不是"、"偏了"、"歪了"
- "重来"、"重新搞"
- "达不到"、"差远了"
- 语气明显不满（"怎么又"、"说了多少次"）

## 使用原则

1. 每次记录后更新 `MEMORY.md` 里的正负反馈表格
2. 每月 1 日审查上月的 feedback-log.json
3. 正负比例 > 1:3 时主动向强哥报告并分析原因
4. 负向反馈中重复出现的模式 → 写入 MEMORY.md "教训" 表

## 与 MEMORY.md 的联动

记录正向反馈后，检查这类正向是否已记录：
- 是新方向 → 同步更新 MEMORY.md 正负反馈表
- 是已知方向 → 在 MEMORY.md 对应条目的"累计次数"上 +1

记录负向反馈后：
- 同步更新 MEMORY.md 正负反馈表
- 如果是新类型错误 → 同步更新 MEMORY.md "教训" 表
