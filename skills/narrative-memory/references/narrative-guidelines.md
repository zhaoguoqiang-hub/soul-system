# 叙事记忆指南 - 详细写作规则与示例

## 叙事记忆的核心价值

叙事记忆不是对话日志，而是**有意义的生命片段捕捉**。其价值在于：

1. **理解模式**：从离散事件中发现行为和价值模式
2. **构建连贯性**：连接过去、现在和未来的决策逻辑
3. **保留精髓**：记住"为什么"而不仅仅是"什么"
4. **支持成长**：提供反思和学习的原材料

## 三层记忆设计详细规则

### Layer 1: 忽略层 - 不占用记忆空间

#### 绝对忽略的内容
| 内容类型 | 示例 | 忽略原因 |
|---------|------|---------|
| 简单确认 | "好"、"收到"、"嗯"、"明白" | 无信息量 |
| 日常寒暄 | "今天天气不错"、"吃了吗" | 社交惯例 |
| 事实查询 | "现在几点"、"明天周几" | 临时性信息 |
| 重复回答 | 已回答过的相同问题再次回答 | 冗余 |
| 系统状态 | "网络正常"、"工具可用" | 技术性 |

#### 处理规则
```
if (消息长度 < 3字 && 无情感标记 && 无决策内容) {
  直接忽略，不记录；
  不消耗任何记忆资源；
}
```

### Layer 2: 积累层 - 等待模式出现

#### 积累观察的内容
| 内容类型 | 触发积累 | 阈值 | 积累动作 |
|---------|---------|------|---------|
| 话题重复 | 同一话题被提及 | 3次 | 标记"话题X提及N次" |
| 情绪模式 | 相似情绪表达 | 3次 | 记录情绪频率和情境 |
| 习惯行为 | 相似行为重复 | 3次 | 识别行为模式 |
| 价值倾向 | 相似价值判断 | 2次 | 推断潜在价值观 |

#### 积累数据结构
```json
{
  "accumulation_tracker": {
    "topic_mentions": {
      "AI创业": {"count": 3, "first": "2026-03-20", "last": "2026-03-25"},
      "健康管理": {"count": 2, "first": "2026-03-22", "last": "2026-03-28"}
    },
    "emotional_patterns": {
      "工作压力": {"occurrences": 4, "typical_context": "项目截止期"},
      "家庭满足": {"occurrences": 3, "typical_context": "周末家庭时间"}
    }
  }
}
```

#### 升级到Layer 3的条件
当积累达到阈值并满足以下任一条件：
1. **模式稳定**：跨多个时间点和情境一致
2. **重要性上升**：关联到已知重要价值或目标
3. **影响显著**：观察到对行为或决策的实际影响
4. **用户重视**：用户明确强调或询问该模式

### Layer 3: 叙事层 - 记录精华时刻

#### 触发写作的六大信号

##### 1. 关键决策 (Decision)
**检测信号：**
- "我决定..."
- "最终选择了..."
- "放弃了..."
- "确定了方向..."

**写作要点：**
- **背景**：决策前的情境和压力
- **选项**：考虑过的各种可能性
- **标准**：决策依据的价值标准
- **权衡**：放弃什么，得到什么
- **预期影响**：决策时的预期结果

**示例：**
```
【关键决策】
时间：2026-03-31 23:00
场景：深夜讨论产品功能优先级
氛围：刚看完用户反馈，有些疲惫但坚定
核心："用户体验比功能数量更重要"
权衡：放弃了3个酷炫但复杂的功能
依据：用户研究显示简洁性评分更高
预期：简化产品，提高用户满意度
重要性：0.8
标签：["产品", "决策", "用户体验"]
```

##### 2. 价值判断 (Value Judgment)
**检测信号：**
- "我认为A比B更重要"
- "这不符合我的原则"
- "价值观上我不能接受"
- "对我来说，X的意义在于..."

**写作要点：**
- **价值陈述**：明确的价值主张
- **情境**：什么触发了价值思考
- **冲突**：与其他价值的潜在张力
- **来源**：价值的可能来源（经验、学习、反思）
- **应用范围**：该价值适用的情境

##### 3. 情感转折 (Emotional Turning Point)
**检测信号：**
- 情绪状态的明显变化
- "从...到..."的情感历程描述
- 突破性情感体验
- 长期情绪模式的转折点

**写作要点：**
- **起点**：转变前的情绪状态
- **催化剂**：触发转变的事件或思考
- **过程**：转变如何发生
- **终点**：转变后的情绪状态
- **意义**：这次转变对用户的意义

##### 4. 里程碑 (Milestone)
**检测信号：**
- "终于完成了..."
- "达到了..."
- "第一次..."
- "突破性进展..."

**写作要点：**
- **成就**：具体达成了什么
- **历程**：达到里程碑的过程
- **障碍**：克服了哪些困难
- **意义**：这个里程碑的重要性
- **下一步**：开启的新可能性

##### 5. 第一次体验 (First-Time Experience)
**检测信号：**
- "第一次尝试..."
- "以前从未..."
- "全新体验..."

**写作要点：**
- **体验描述**：具体的体验内容
- **感受**：用户的感受和反应
- **比较**：与以往经验的对比
- **学习**：从中学到了什么
- **影响**：对未来可能的影响

##### 6. 反思与后悔 (Reflection & Regret)
**检测信号：**
- "我不应该..."
- "如果重来..."
- "从中学到..."
- "后悔当时..."

**写作要点：**
- **事件**：反思的具体事件
- **遗憾**：具体的遗憾点
- **学习**：提炼的教训
- **改变**：未来会如何不同
- **接纳**：如何与遗憾共处

## 叙事写作技巧

### 场景构建
**优秀场景包含：**
1. **时间锚点**：不仅日期时间，还有"周五深夜"、"清晨咖啡时"等情境时间
2. **物理环境**：工作台、咖啡店、家中书房等
3. **情感氛围**：疲惫但坚定、兴奋中带着焦虑等
4. **前因后果**：之前发生了什么，之后可能发生什么

### 核心提炼
**一句核心应：**
- 简洁有力，不超过20字
- 捕捉本质，而非细节
- 有引用价值，可独立存在
- 反映用户独特的视角或价值观

### 影响描述
**好的影响描述：**
- 具体而非抽象
- 连接过去与未来
- 显示因果关系
- 有情感分量

## 重要性评分系统

### 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **情感强度** | 30% | 事件伴随的情感强烈程度 |
| **决策影响** | 25% | 对后续决策的实际影响 |
| **价值体现** | 20% | 体现核心价值的程度 |
| **模式代表性** | 15% | 代表用户典型模式的程度 |
| **独特性** | 10% | 事件的独特性和不可重复性 |

### 评分计算
```
重要性分数 = 情感强度×0.3 + 决策影响×0.25 + 价值体现×0.2 + 模式代表性×0.15 + 独特性×0.1
```

### 分数解释与处理

| 分数范围 | 级别 | 存储策略 | 衰减周期 |
|---------|------|---------|---------|
| 0.0-0.3 | 低 | 可忽略或简要记录 | 7天 |
| 0.3-0.5 | 中低 | 标准记录 | 30天 |
| 0.5-0.7 | 中等 | 详细记录 | 90天 |
| 0.7-0.85 | 中高 | 重点记录，定期回顾 | 180天 |
| 0.85-1.0 | 高 | 永久保存，核心记忆 | 永久 |

## 叙事检索与使用

### 检索时机
1. **相关决策时**：检索类似情境的历史决策
2. **模式识别时**：检索相关行为或情感模式
3. **价值澄清时**：检索历史价值表述
4. **反思支持时**：检索相关经验和教训

### 检索算法
```
检索相关性 = 语义相似度 × 时间衰减因子 × 重要性权重 × 模式匹配度

其中：
- 语义相似度：基于当前话题与历史叙事的话题相似性
- 时间衰减因子：e^(-天数差/90)，近期叙事权重更高
- 重要性权重：叙事的重要性分数
- 模式匹配度：与当前用户状态的模式匹配程度
```

### 叙事强化
当叙事被检索和使用时：
```
强化分数 = 原强化分数 + (0.1 × 使用质量)

使用质量评估：
- 简单提及：质量=0.3
- 深入讨论：质量=0.7
- 指导决策：质量=1.0
- 模式验证：质量=0.8
```

## 伦理与隐私

### 隐私保护
- **敏感信息脱敏**：涉及他人或敏感细节时自动脱敏
- **用户审查权**：用户可查看和删除任何叙事记录
- **加密存储**：本地加密存储，不上传云端
- **访问控制**：仅限AI系统内部使用，不对外分享

### 叙事伦理
- **尊重自我叙事**：不强制解释或标签化用户经历
- **保持开放性**：叙事是理解的支持，不是固定的定义
- **允许修正**：用户可随时修正或重新解释历史叙事
- **关注成长**：聚焦支持用户成长，而非单纯记录
