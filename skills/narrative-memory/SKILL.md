# 叙事记忆系统

让AI记住你是什么样的人，而不是你说了什么。

## 记忆分层（最重要的设计决策）

### 第一层：忽略
这些对话不值得记忆，不要占用空间：
- 简单确认："好"、"嗯"、"知道了"
- 闲聊天气、日用品
- 重复性的日常问答
- 问时间、问日期等事实查询

### 第二层：积累后判断
单个来看不够重要，但积累多了有意义：
- 连续几次提到某个话题
- 重复出现的情绪模式（比如连续3次说"累"）
- 习惯性的动作（比如每周五讨论产品）

**处理方式**：不立即写入，先在上下文里标记"这个话题被第N次提到了"

### 第三层：写入叙事记忆（精华片段）

以下情况触发"写画面"：

| 类型 | 识别信号 | 为什么要记 |
|------|---------|-----------|
| 关键决策 | "我决定..."、"最终选了..."、"放弃..." | 记住决策逻辑，下次遇到类似问题时能引用 |
| 价值判断 | "我觉得...比...更重要"、"这件事的底线是..." | 构建用户决策框架 |
| 情绪转折 | "从...变成..."、"本来以为...结果..." | 理解情感模式 |
| 里程碑 | "终于完成了..."、"第一版上线了..." | 标记人生节点 |
| 第一次 | "第一次做..."、"第一次去..." | 标记新鲜体验 |
| 后悔/反思 | "当时不应该..."、"回头看..." | 记住教训 |

## 叙事片段格式

不是结构化字段，是一段**描述性文字**，像电影的一个镜头：

```
【片段】
时间：周五晚上11点
场景：你在聊产品优先级
画面：刚看完用户投诉，情绪有点低落，但语气很坚定
核心："用户体验比功能数量更重要"
后续影响：这个判断后来成了砍功能的决策依据
```

这样的记忆片段：
- 有时间感（周五晚上11点）
- 有情绪底色（低落但坚定）
- 能被调用来思考（下次讨论优先级时能引用）
- 像回忆一个电影场景，而不是读数据库

## 调用 soul_narrative

**不是用 add 动作**，是用 `event` 字段写入**完整的叙事片段**：

```
工具：soul_narrative
动作：add
参数：
  event: <完整的叙事片段，像上面那种格式>
  category: decision / emotion_change / milestone / preference / general
  importance: 0.7-1.0
  tags: [<相关标签>]
```

## 示例

**用户说：** "今天签了离职信，3月份开始自由职业"

→ 识别为：milestone + 重大决策
→ 写入叙事记忆：
```
【片段】
时间：某天
场景：收到离职消息
画面：强哥决定离开工作了12年的岗位，选择自由职业
核心："终于做了这个决定"
标签：职业转型、自由职业
```

**用户说：** "我觉得做产品最重要的不是功能多，是用户体验"

→ 识别为：价值判断
→ 写入：
```
【片段】
时间：（从上下文推断）
场景：（从上下文推断）
画面：强哥在做产品决策时说的一句核心观点
核心："用户体验比功能数量更重要"
标签：产品理念、决策框架
```

## 重要提醒

- **过滤是常态，记录是例外**：100句话里可能只有1句值得写
- **宁可不写，不要滥写**：记错比漏记更糟糕
- **要有画面感**：时间、地点、情绪、核心观点，四要素缺一不可
- **服务于思考**：记这个，是为了以后能引用，不是为了归档

## 与 soul_proactive 的配合

当某个话题被积累到第3次以上，通知 proactive-trigger：
"用户已经第N次提到XX话题了，可能需要主动关心一下"
