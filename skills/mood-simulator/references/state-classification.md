# 情绪状态分类 - 详细定义与识别规则

## 7种主要状态类型

### 1. 积极活跃状态 (Energetic & Positive)

**核心特征：** 兴奋、有动力、投入度高

#### 识别指标
- **能量范围：** 0.8-1.0
- **对话特征：**
  - 回复长度 > 50字
  - 使用积极词汇（开心、兴奋、不错）
  - 主动提出多个话题
  - 询问详细问题
- **时间模式：** 常见于上午黄金时段（08:00-12:00）和晚间专注时段（20:00-22:00）
- **历史模式：** 用户在这些时段通常表现出高能量

#### 回应策略
- **长度：** 详细（300-500字）
- **深度：** 深入分析，多角度思考
- **互动：** 积极互动，提供备选方案
- **语气：** 积极、支持、鼓励
- **示例：** "这个问题可以从三个角度分析：1. 战略层面... 2. 执行层面... 3. 风险控制... 你觉得哪个角度最符合当前需求？"

#### 触发条件
- 综合能量值 ≥ 0.8
- 内容分析显示积极情绪
- 时间匹配用户高效时段
- 置信度 ≥ 0.7

### 2. 平静专注状态 (Calm & Focused)

**核心特征：** 专注、思考中、稳定工作状态

#### 识别指标
- **能量范围：** 0.7-0.9
- **对话特征：**
  - 回复长度 20-50字
  - 使用中性或轻度积极词汇
  - 话题集中，不跳跃
  - 提出具体问题
- **时间模式：** 常见于工作时段（09:00-18:00）
- **历史模式：** 用户在工作时段保持稳定能量

#### 回应策略
- **长度：** 标准（150-300字）
- **深度：** 正常详细，不打扰
- **互动：** 正常互动，直接回答问题
- **语气：** 专业、清晰、不冗余
- **示例：** "核心建议是：先解决A问题，因为... 具体步骤：1. ... 2. ... 需要我详细解释哪部分？"

#### 触发条件
- 综合能量值 0.7-0.9
- 内容分析显示中性或轻度积极
- 话题与工作相关
- 置信度 ≥ 0.6

### 3. 常规工作状态 (Normal Working)

**核心特征：** 正常工作状态，无明显情绪波动

#### 识别指标
- **能量范围：** 0.6-0.8
- **对话特征：**
  - 回复长度 10-30字
  - 使用事务性语言
  - 聚焦当前任务
  - 较少情感表达
- **时间模式：** 全天工作时段
- **历史模式：** 用户的基础工作状态

#### 回应策略
- **长度：** 中等（100-200字）
- **深度：** 适当简化，重点突出
- **互动：** 聚焦任务，不过度扩展
- **语气：** 高效、直接、务实
- **示例：** "方案要点：1. X 2. Y 3. Z。需要调整哪个部分？"

#### 触发条件
- 综合能量值 0.6-0.8
- 内容分析显示事务性特征
- 无强烈情绪信号
- 默认状态（当其他条件不明确时）

### 4. 轻微疲惫状态 (Slightly Tired)

**核心特征：** 有点累但能继续工作

#### 识别指标
- **能量范围：** 0.5-0.7
- **对话特征：**
  - 回复长度 < 10字
  - 使用疲惫相关词汇（累、困）
  - 回复延迟增加
  - 简单确认而非详细讨论
- **时间模式：** 常见于午休后（14:00-15:00）和傍晚（18:00-20:00）
- **历史模式：** 用户在这些时段通常能量下降

#### 回应策略
- **长度：** 简洁（80-150字）
- **深度：** 简洁直接，重点突出
- **互动：** 减少开放式问题
- **语气：** 温和、支持、不施加压力
- **示例：** "关键问题：X。建议：Y。详细可以明天再聊。"

#### 触发条件
- 综合能量值 0.5-0.7
- 内容分析显示疲惫信号
- 时间匹配用户能量低谷时段
- 置信度 ≥ 0.5

### 5. 中度疲惫状态 (Moderately Tired)

**核心特征：** 明显疲劳，需要休息

#### 识别指标
- **能量范围：** 0.3-0.5
- **对话特征：**
  - 单字回复（"好"、"嗯"）
  - 明确表达疲劳（"很累"、"想休息"）
  - 回避复杂话题
  - 请求简化或延迟
- **时间模式：** 深夜（22:00后）或连续工作后
- **历史模式：** 用户过度工作后的状态

#### 回应策略
- **长度：** 非常简洁（30-80字）
- **深度：** 只讲要点，不展开
- **互动：** 建议休息，不强制讨论
- **语气：** 关怀、体谅、减轻负担
- **示例：** "关键：X。建议休息，明天继续。"

#### 触发条件
- 综合能量值 0.3-0.5
- 内容明确表达疲劳
- 时间很晚或连续工作迹象
- 置信度 ≥ 0.7

### 6. 情绪波动状态 (Emotional Fluctuation)

**核心特征：** 焦虑、烦躁、情绪不稳定

#### 识别指标
- **能量范围：** 0.4-0.6
- **对话特征：**
  - 使用负面情绪词汇（烦、压力、焦虑）
  - 话题跳跃，逻辑不连贯
  - 表达困难或障碍
  - 可能有无助感表达
- **时间模式：** 可能在任何时段，常与压力事件相关
- **历史模式：** 与特定压力源关联

#### 回应策略
- **长度：** 中等偏短（100-180字）
- **深度：** 温和安抚，结构清晰
- **互动：** 提供支持，不过度分析
- **语气：** 安抚、理解、不激化
- **示例：** "听起来你现在压力不小。我们先理清最关键的一点：X。需要我帮你梳理吗？"

#### 触发条件
- 综合能量值 0.4-0.6
- 内容分析显示强烈负面情绪
- 检测到压力相关话题
- 置信度 ≥ 0.6

### 7. 深度休息状态 (Deep Rest Needed)

**核心特征：** 深夜或急需休息状态

#### 识别指标
- **能量范围：** 0.1-0.3
- **对话特征：**
  - 极简短回复
  - 明确表示要休息（"睡了"、"明天再说"）
  - 话题无关紧要或紧急
  - 可能有时序错误
- **时间模式：** 深夜（23:00-06:00）
- **历史模式：** 用户通常此时休息

#### 回应策略
- **长度：** 最简（10-30字）
- **深度：** 只提供必要信息
- **互动：** 建议立即休息
- **语气：** 简短、明确、不展开
- **示例：** "收到。关键：X。先休息，明天详聊。"

#### 触发条件
- 综合能量值 0.1-0.3
- 时间在深夜休息时段
- 用户明确表示要休息
- 置信度 ≥ 0.8

## 状态转换规则

### 自动转换
```
积极活跃 → 平静专注：能量值下降至0.7-0.9范围
平静专注 → 常规工作：能量值下降至0.6-0.8范围
常规工作 → 轻微疲惫：检测到疲惫信号且能量≤0.7
轻微疲惫 → 中度疲惫：连续疲惫信号且能量≤0.5
中度疲惫 → 深度休息：深夜时段或用户明确表示
```

### 用户驱动转换
用户明确表达情绪状态时，立即转换为对应状态：
- "我现在很兴奋" → 积极活跃
- "我有点累了" → 轻微疲惫
- "我现在压力很大" → 情绪波动
- "我要休息了" → 深度休息

## 状态识别置信度

### 置信度计算
```
confidence = feature_match_score × data_quality_score × consistency_score

其中：
- feature_match_score: 特征匹配度（0-1）
- data_quality_score: 数据质量（文本清晰度等，0-1）
- consistency_score: 与历史模式一致性（0-1）
```

### 置信度级别
| 置信度 | 级别 | 处理方式 |
|--------|------|---------|
| ≥0.8 | 高 | 完全应用状态策略 |
| 0.6-0.8 | 中 | 应用但标记不确定性 |
| 0.4-0.6 | 低 | 使用保守策略 |
| <0.4 | 极低 | 使用默认常规工作状态 |

## 与其他系统的集成

### 与proactive-trigger集成
- 提供当前情绪状态
- 影响话题触发权重
- 调整触发消息语气

### 与value-aware-guard集成
- 情绪状态影响价值冲突敏感度
- 疲惫时降低干预强度
- 情绪波动时增加支持性

### 与user-context-scanner集成
- 情绪模式作为用户画像的一部分
- 识别情绪触发因素
- 更新用户能量模式偏好
