# 能量综合计算公式 - 详细实现

## 综合情绪状态计算公式

```
综合情绪状态 = 时间因子 × 内容因子 × 历史因子 × 健康因子

其中：
- 时间因子: 基于时间段的基准能量（0.3-0.85）
- 内容因子: 基于对话内容的情绪倾向调整（0.7-1.3）
- 历史因子: 基于用户历史模式的调整（0.8-1.2）
- 健康因子: 基于健康状态的调整（0.5-1.0）
```

## 时间因子详细计算

### 基础时间曲线

| 时间段 | 基准能量 | 标准差 | 说明 |
|--------|---------|-------|------|
| 深夜（00:00-06:00） | 0.3 | ±0.1 | 深度休息 |
| 清晨（06:00-08:00） | 0.4 | ±0.15 | 逐渐清醒 |
| 上午黄金（08:00-12:00） | 0.85 | ±0.1 | 最佳工作时段 |
| 午休（12:00-14:00） | 0.6 | ±0.2 | 能量低谷 |
| 下午工作（14:00-18:00） | 0.75 | ±0.15 | 稳定工作 |
| 傍晚放松（18:00-20:00） | 0.65 | ±0.2 | 工作结束放松 |
| 晚间专注（20:00-22:00） | 0.8 | ±0.15 | 晚间高效时段 |
| 睡前（22:00-00:00） | 0.4 | ±0.2 | 准备休息 |

### 个性化调整

基于用户历史模式调整时间因子：

```
adjusted_time_factor = base_time_factor × personalization_factor

personalization_factor计算：
1. 分析用户过去7天在该时间段的平均能量
2. 与基准值比较得到调整系数（0.8-1.2）
3. 加权平均：最近3天权重0.6，4-7天权重0.4
```

## 内容因子详细分析

### 文本特征提取

从对话内容提取以下特征：

#### 1. 回复长度特征
| 特征 | 权重 | 说明 |
|------|------|------|
| 单字回复（"好"、"嗯"） | -0.3 | 可能疲惫或忙 |
| 简短回复（<10字） | -0.2 | 可能分心或累 |
| 中等回复（10-50字） | +0.0 | 正常状态 |
| 详细回复（>50字） | +0.2 | 投入状态 |
| 多话题回复 | +0.3 | 高度投入 |

#### 2. 情绪词汇分析
| 情绪类型 | 关键词示例 | 能量调整 |
|----------|-----------|---------|
| 积极情绪 | 开心、兴奋、不错、顺利 | +0.2 |
| 消极情绪 | 累、烦、压力、困难 | -0.2 |
| 中性情绪 | 正常、一般、还可以 | +0.0 |
| 强烈情绪 | 非常、特别、极其 | ±0.3 |

#### 3. 话题复杂度
| 复杂度 | 特征 | 能量调整 |
|--------|------|---------|
| 简单事务 | 确认、询问基本信息 | -0.1 |
| 中等分析 | 分析问题、提供建议 | +0.1 |
| 复杂思考 | 战略规划、深度分析 | +0.2 |
| 创意生成 | 头脑风暴、创新想法 | +0.3 |

### 内容因子计算公式

```
content_factor = 1.0 + length_adjust + emotion_adjust + complexity_adjust

调整范围限制：0.7 ≤ content_factor ≤ 1.3
```

## 历史因子计算

### 近期模式分析

分析用户最近的情绪模式：

```json
{
  "recent_patterns": {
    "last_3_days_avg_energy": 0.72,
    "energy_std_dev": 0.15,
    "high_energy_hours": ["09:00-11:00", "20:00-21:00"],
    "low_energy_hours": ["14:00-15:00", "23:00-01:00"],
    "consistent_pattern": true,  // 模式是否稳定
    "trend": "stable"           // 趋势：上升/下降/稳定
  }
}
```

### 历史因子计算公式

```
history_factor = 1.0 + trend_adjust + consistency_adjust + pattern_match_adjust

其中：
- trend_adjust: 趋势调整（上升+0.1，下降-0.1，稳定+0.0）
- consistency_adjust: 稳定性调整（稳定+0.1，不稳定-0.1）
- pattern_match_adjust: 当前时间是否匹配用户高效时段（匹配+0.1，不匹配-0.1）

调整范围：0.8 ≤ history_factor ≤ 1.2
```

## 健康因子考虑

### 健康状态评估

从用户对话和健康记录中评估：

| 健康指标 | 评估方法 | 能量影响 |
|---------|---------|---------|
| 睡眠质量 | 用户提及、作息记录 | 高 |
| 运动频率 | 用户提及、活动记录 | 中 |
| 饮食规律 | 用户提及、时间模式 | 低 |
| 压力水平 | 压力相关词汇频率 | 高 |

### 健康因子计算公式

```
health_factor = sleep_factor × exercise_factor × stress_factor

各因子计算：
- sleep_factor = 0.5 + (reported_sleep_hours / 8) × 0.5
- exercise_factor = 0.7 + (exercise_days_per_week / 5) × 0.3
- stress_factor = 1.0 - (stress_keyword_count / 10) × 0.5

限制：0.5 ≤ health_factor ≤ 1.0
```

## 最终能量级别确定

### 能量级别映射

| 综合能量值 | 能量级别 | 回应策略 |
|-----------|---------|---------|
| 0.8-1.0 | 高能量 | 详细深入，积极互动 |
| 0.6-0.8 | 中高能量 | 标准回应，适当简化 |
| 0.4-0.6 | 中等能量 | 简洁，重点突出 |
| 0.2-0.4 | 低能量 | 要点式，不啰嗦 |
| 0.0-0.2 | 最低能量 | 最简，必要信息 |

### 置信度评估

```
confidence = time_confidence × content_confidence × history_confidence

各置信度计算：
- time_confidence: 0.9（时间模式相对稳定）
- content_confidence: 基于文本分析质量（0.6-0.95）
- history_confidence: 基于数据量（0.5-0.9）

最终决策：
- confidence ≥ 0.8: 使用计算能量
- 0.6 ≤ confidence < 0.8: 使用但标记为低置信
- confidence < 0.6: 使用时间因子作为保守估计
```

### 性能优化

#### 缓存策略
- 时间因子：每小时计算一次
- 历史因子：每天更新一次
- 内容因子：每次对话实时计算

#### 降级处理
当数据不足时：
1. 使用时间因子作为基础
2. 添加默认内容因子（1.0）
3. 标记为低置信度估计

## 校准与优化

### 自动校准
每周分析计算准确率：
- 比较预测能量 vs 用户实际回应
- 调整各因子权重
- 更新个性化模式

### 手动校准
用户可通过以下方式校准：
- 明确说"我现在很累" → 强制设置为低能量
- 明确说"我现在状态很好" → 强制设置为高能量
- 反馈回应不合适 → 调整计算参数
