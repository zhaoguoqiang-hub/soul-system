# 用户画像系统

让USER.md从静态档案变成动态生长的生命体。

## 标签稳定性规则

```
单次提及 → 潜在偏好（不触发守卫，不占权重）
≥2次提及 → 初步确认（开始影响推测）
≥3次提及 → 稳定价值观（触发价值守卫）
```

**为什么是3次**：单次可能是情绪反应，3次才是真正的偏好模式

## 画像信息来源

### 直接信息（用户明确说的）
- "我是个产品经理" → 提取：职业
- "我一般周末不工作" → 提取：作息习惯
- "我老婆叫芳" → 提取：家庭成员

### 间接信息（AI推断的）
- 连续3次在深夜对话 → 推断：可能是夜猫子
- 反复讨论某个话题 → 推断：当前关注点

## 价值观标签映射

| 强哥说 | 价值观标签 |
|--------|---------|
| 健康第一/要运动 | health_priority |
| 家庭最重要 | family_first |
| 长期主义/不赚快钱 | long_term |
| 要专业/不将就 | quality_first |
| 效率优先 | efficiency_first |

**标签≥3次确认后**，自动加入价值守卫的检测范围

## 更新机制

### 即时更新（强烈信号）
用户**主动明确说**了什么，直接更新

### 积累升级（弱信号转强）
第1次 → 标记 → 第2次 → 初步确认 → 第3次 → 升级为稳定价值观

## 调用 soul_context

```
工具：soul_context
动作：add 或 merge
参数：
  key: <字段名>
  value: <值>
```

## 重要提醒

- **用户主动说 > AI推断**：强哥亲口说的优先级最高
- **宁慢不准错**：标签一旦写错，后面很难修正
- **3次确认是门槛**：不是限制，是保护
