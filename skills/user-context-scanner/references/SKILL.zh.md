# 用户画像系统

让USER.md从静态档案变成动态生长的生命体。

## 架构

**独立模式**：直接写入USER.md
**有插件时**：使用 soul_context 工具

## 标签稳定性规则

```
单次提及 → 潜在偏好（不触发守卫）
≥2次提及 → 初步确认（开始影响推测）
≥3次提及 → 稳定价值观（触发价值守卫）
```

## 画像信息来源

**直接信息**：用户明确说的
**间接信息**：AI推断的（需3次确认才生效）

## 更新机制

- 即时更新（强烈信号）：直接写入USER.md
- 积累升级：1次→2次→3次→升级为稳定价值观

## 调用 soul_context（有插件时）

```
工具：soul_context
动作：add 或 merge
参数：
  key: <字段名>
  value: <值>
```

**无需任何依赖，独立模式即可工作**
