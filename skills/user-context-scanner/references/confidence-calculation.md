# 置信度计算详细规则 - 量化评估画像准确性

## 置信度计算核心公式

### 基础公式
```
confidence_score = source_credibility × evidence_strength × consistency_factor

其中：
- source_credibility ∈ [0, 1]: 信息来源可信度
- evidence_strength ∈ [0, 1]: 证据强度
- consistency_factor ∈ [0, 1]: 一致性因子
```

### 各分量详细计算

#### 1. 信息来源可信度 (source_credibility)

基于信息来源类型和质量的权重评估：

| 来源类型 | 基础可信度 | 说明 | 调整因素 |
|---------|-----------|------|---------|
| **用户明确陈述** | 0.95 | 用户直接说出的信息 | ±0.05（基于陈述确定性） |
| **重复行为模式** | 0.85 | 可观察的重复行为（≥3次） | +0.1（跨情境一致性） |
| **单次行为观察** | 0.65 | 单次观察到的行为 | ±0.15（情境典型性） |
| **上下文推断** | 0.55 | 从对话中逻辑推断 | ±0.2（推断合理性） |
| **外部数据源** | 0.70 | 传感器、日历、应用数据 | ±0.1（数据质量） |
| **AI推测** | 0.45 | 无直接证据的模型推测 | ±0.25（模型置信度） |

**可信度调整规则：**
- **陈述确定性：** 用户使用"绝对"、"肯定"等词语 → +0.05
- **情境典型性：** 行为在典型情境下发生 → +0.1
- **数据完整性：** 外部数据完整无缺失 → +0.05
- **推断合理性：** 推断逻辑严密 → +0.1

#### 2. 证据强度 (evidence_strength)

评估单个证据的质量和可靠性：

| 证据特征 | 强度系数 | 说明 | 计算公式 |
|---------|---------|------|---------|
| **时间衰减** | 0.4-1.0 | 证据新鲜度影响 | 1.0 × e^(-天数/30) |
| **具体程度** | 0.6-1.0 | 细节丰富程度 | 基础 + 细节分数×0.2 |
| **多源验证** | 1.0-1.3 | 多个独立来源支持 | 1.0 + 0.1×验证源数 |
| **情感强度** | 0.8-1.2 | 伴随的情感表达强度 | 1.0 ± 情感强度×0.2 |
| **情境一致性** | 0.7-1.1 | 与典型情境的匹配度 | 1.0 ± 匹配度×0.1 |

**时间衰减详细计算：**
```
time_decay_factor = e^(-days_since_evidence / half_life)

其中：
- days_since_evidence: 证据距今天数
- half_life: 半衰期（默认30天）
- e: 自然常数
```

**具体程度评分标准：**
- **高具体性（+0.2）：** 包含时间、地点、人物、原因、过程
- **中具体性（+0.1）：** 包含3-4个具体要素
- **低具体性（0）：** 模糊描述，缺乏细节

#### 3. 一致性因子 (consistency_factor)

评估证据与现有画像的一致性程度：

```
consistency_factor = 1.0 - (contradiction_weight / total_weight) × severity_factor

其中：
- contradiction_weight: 矛盾证据总权重
- total_weight: 所有相关证据总权重
- severity_factor: 矛盾严重性系数（0.5-1.0）
```

**矛盾严重性系数：**
| 矛盾类型 | 严重性系数 | 说明 |
|---------|-----------|------|
| 逻辑矛盾 | 0.9 | 直接逻辑冲突 |
| 价值矛盾 | 0.8 | 核心价值不一致 |
| 行为矛盾 | 0.7 | 行为模式不一致 |
| 情境差异 | 0.6 | 不同情境下的合理差异 |
| 时间演变 | 0.5 | 随时间变化的合理调整 |

### 综合置信度计算示例

**场景：** 推断用户"喜欢早上工作"

**证据：**
1. 用户说："我早上效率最高"（3天前）
2. 观察到3次在09:00-11:00处理复杂任务（最近7天）
3. 用户曾在压力下说："晚上完全没法思考"（15天前）

**计算过程：**
```
source_credibility = 
  证据1：0.95（明确陈述）× 0.9（时间衰减3天） = 0.855
  证据2：0.85（行为模式）× 1.1（多源3次） = 0.935
  证据3：0.65（单次行为）× 0.7（时间衰减15天） = 0.455

evidence_strength = 
  证据1：0.9（时间）× 0.8（具体性） = 0.72
  证据2：1.0（时间）× 0.9（具体性）× 1.2（多源） = 1.08
  证据3：0.7（时间）× 0.6（具体性） = 0.42

consistency_factor = 1.0（无矛盾证据）

综合置信度 = 
  (0.855×0.72 + 0.935×1.08 + 0.455×0.42) / (0.72+1.08+0.42) × 1.0
  = (0.6156 + 1.0098 + 0.1911) / 2.22
  = 1.8165 / 2.22
  = 0.818
```

## 置信度级别解释与应用

### 置信度分级标准

| 置信度范围 | 级别 | 颜色标识 | 使用建议 |
|-----------|------|---------|---------|
| 0.90-1.00 | 极高置信 | 🟢 绿色 | 可作为稳定事实，无需验证 |
| 0.75-0.90 | 高置信 | 🟡 黄色 | 可信任使用，偶尔抽样验证 |
| 0.60-0.75 | 中等置信 | 🟠 橙色 | 谨慎使用，建议定期验证 |
| 0.40-0.60 | 低置信 | 🔴 红色 | 仅作参考，需要主动验证 |
| 0.00-0.40 | 极低置信 | ⚫ 黑色 | 不应写入正式画像 |

### 各级别应用规则

#### 极高置信（0.9-1.0）
**特征：** 多个高质量证据，无矛盾，跨时间一致
**应用：**
- 可作为用户画像的稳定基础
- 用于重要决策支持
- 无需主动验证，除非出现新矛盾
- 定期（每季度）审查即可

#### 高置信（0.75-0.9）
**特征：** 有较好证据支持，可能有轻微不一致
**应用：**
- 可正常使用于个性化交互
- 每1-2个月抽样验证一次
- 注意监控相关新证据
- 发现矛盾时及时降级

#### 中等置信（0.6-0.75）
**特征：** 证据有限或有一定矛盾
**应用：**
- 使用但注明不确定性
- 每月至少验证一次
- 不用于关键决策
- 积极寻找验证机会

#### 低置信（0.4-0.6）
**特征：** 证据薄弱或明显矛盾
**应用：**
- 仅作临时参考
- 需要主动验证（Quiz）
- 不写入长期画像
- 标记为"待验证"

#### 极低置信（<0.4）
**特征：** 几乎无可靠证据
**应用：**
- 不记录，仅作内部参考
- 立即寻求验证
- 如无法验证则放弃
- 用于模式探索而非事实认定

## 置信度动态调整

### 时间衰减机制

**基础衰减公式：**
```
confidence_t = confidence_0 × e^(-λt)

其中：
- confidence_0: 初始置信度
- λ: 衰减系数，基于证据类型
- t: 时间（天）
```

**不同类型证据的衰减系数：**
| 证据类型 | 衰减系数λ | 半衰期 | 说明 |
|---------|----------|-------|------|
| 稳定特质 | 0.001 | 693天 | 性格、核心价值观 |
| 长期习惯 | 0.005 | 139天 | 作息、工作模式 |
| 中期偏好 | 0.01 | 69天 | 爱好、学习方式 |
| 短期兴趣 | 0.03 | 23天 | 当前关注话题 |
| 临时状态 | 0.1 | 7天 | 情绪、临时偏好 |

### 基于新证据的调整

#### 强化调整（新支持证据）
```
new_confidence = old_confidence + (evidence_weight × reinforcement_factor × (1 - old_confidence))

其中：
- evidence_weight: 新证据权重（0-1）
- reinforcement_factor: 强化系数，默认0.3
```

#### 削弱调整（新矛盾证据）
```
new_confidence = old_confidence × (1 - contradiction_weight × severity_factor)

其中：
- contradiction_weight: 矛盾证据权重（0-1）
- severity_factor: 矛盾严重性（0.5-1.0）
```

### 一致性压力调整

当多个相关证据出现不一致时，系统会产生"一致性压力"，推动置信度调整：

```
consistency_pressure = Σ|confidence_i - mean_confidence| / n

调整动作：
if consistency_pressure > 0.3:
   重新评估所有相关证据
   寻找解释或分辨主次
   相应调整各证据置信度
```

## 置信度在画像中的应用

### 字段级置信度

每个画像字段包含独立的置信度评分：

```json
{
  "field": "work_pattern.focus_hours",
  "value": ["09:00-12:00", "14:00-18:00"],
  "confidence": 0.82,
  "confidence_components": {
    "source_credibility": 0.88,
    "evidence_strength": 0.93,
    "consistency_factor": 1.0
  },
  "last_updated": "2026-04-01",
  "next_review": "2026-05-01"
}
```

### 画像整体置信度

```
overall_confidence = Σ(field_confidence × field_importance) / Σ(field_importance)

其中 field_importance 基于：
1. 使用频率：经常被查询或使用的字段权重更高
2. 影响范围：影响多个决策的字段权重更高
3. 用户重视程度：用户明确关注的字段权重更高
```

### 置信度可视化

在控制面板中显示置信度：

```
👤 用户画像（总体置信度：84%）
├── 🟢 职业信息（产品经理） 96%
├── 🟡 工作习惯（上午高效） 82%
├── 🟠 健康习惯（睡眠不足） 68%
└── 🔴 学习偏好（案例优先） 45%（待验证）
```

## 优化与校准

### 置信度校准机制

定期将置信度预测与实际准确性对比：

```
calibration_error = |predicted_confidence - actual_accuracy|

校准动作：
if calibration_error > 0.15:
   调整置信度计算公式权重
   重新评估历史证据
   更新衰减系数
```

### 用户反馈整合

用户直接反馈是最强的校准信号：

| 反馈类型 | 置信度影响 | 处理方式 |
|---------|-----------|---------|
| 明确确认 | +0.2 | 直接提升置信度 |
| 部分确认 | +0.1 | 有限提升，保留不确定性 |
| 模糊回应 | 0 | 保持现状，继续观察 |
| 部分否定 | -0.15 | 降低，寻求澄清 |
| 明确否定 | -0.3 | 大幅降低或删除字段 |

### 系统学习与改进

基于历史数据分析置信度计算效果：

1. **准确率分析**：高置信度字段的实际准确率
2. **错误模式分析**：哪些类型的推断容易出错
3. **证据权重优化**：调整各证据类型的权重
4. **衰减系数调整**：基于实际变化速度优化衰减

每月生成置信度系统性能报告，持续优化计算模型。
