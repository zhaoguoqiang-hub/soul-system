# 矛盾检测规则 - 详细算法与处理策略

## 矛盾类型定义

系统检测四种主要矛盾类型，每种有不同的处理策略。

### 1. 言行不一致
用户表达的价值、承诺或偏好与实际行为不一致。

#### 典型模式
- **价值声明 vs 行为**："健康第一"但经常熬夜
- **目标声明 vs 行动**："要学习英语"但无学习行为
- **偏好表达 vs 选择**："喜欢简洁"但选择复杂方案

#### 检测算法
```
if (用户明确陈述价值V && 检测到行为B && B与V冲突) {
  矛盾分数 = 价值重要性 × 行为冲突程度 × 频率因子；
  标记为言行不一致；
}
```

### 2. 前后不一致
用户在不同时间对同一事物的表述不一致。

#### 典型模式
- **时间点A vs 时间点B**：昨天说"喜欢"，今天说"不喜欢"
- **不同情境不一致**：工作中说X，家庭中说Y
- **渐进式变化未说明**：偏好逐渐改变但未明确表达

#### 检测算法
```
if (陈述S1时间T1 && 陈述S2时间T2 && S1与S2矛盾 && T2-T1 < 时间窗口) {
  矛盾分数 = 陈述确定性差异 × 时间接近度；
  标记为前后不一致；
}
```

### 3. 情境性矛盾
用户行为在不同情境下表现不一致，但有合理原因。

#### 典型模式
- **工作 vs 家庭**：工作中果断，家庭中犹豫
- **压力 vs 放松**：压力下行为改变
- **公开 vs 私下**：公开表现与私下不同

#### 处理原则
情境性矛盾不一定需要"解决"，而是需要理解和记录上下文。

### 4. 认知失调
用户同时持有两个矛盾的信念或态度。

#### 典型模式
- **价值冲突未察觉**：既想要A又想要B，但A和B冲突
- **行为合理化**：做X但用Y理由解释，X和Y逻辑不一致
- **选择性注意**：只关注支持自己选择的信息

## 矛盾检测详细算法

### 数据源整合

#### 1. 明确陈述提取
从对话中提取用户明确的价值、偏好、目标声明：
- 使用模式匹配："我认为X"、"我相信X"、"我重视X"
- 提取声明内容、确定性程度、情感强度
- 记录上下文和时机

#### 2. 行为模式分析
从互动记录分析实际行为模式：
- 时间分配模式
- 选择倾向性
- 重复行为频率
- 回避或倾向模式

#### 3. 情境因素记录
记录可能影响一致性的情境因素：
- 压力水平
- 时间紧迫性
- 社会环境
- 情绪状态

### 矛盾分数计算

```
contradiction_score = base_conflict × importance_weight × frequency_factor × recency_factor

其中：
- base_conflict: 基础冲突程度（0-1）
- importance_weight: 涉及价值的重要性（1-10，归一化）
- frequency_factor: 矛盾发生频率调整（1 + log10(发生次数)）
- recency_factor: 时间衰减，最近矛盾权重更高
```

#### 基础冲突程度计算

**对于言行不一致：**
```
base_conflict = 1 - (支持行为权重 / 矛盾行为权重)

支持行为权重 = Σ(支持行为强度 × 证据质量)
矛盾行为权重 = Σ(矛盾行为强度 × 证据质量)
```

**对于前后不一致：**
```
base_conflict = 语句相似度 × (1 - 时间衰减因子)

语句相似度：基于语义分析的陈述相似度（0-1）
时间衰减因子：e^(-时间差/30)  // 30天半衰期
```

### 置信度评估

```
confidence = evidence_quality × source_reliability × contextual_clarity

其中：
- evidence_quality: 证据质量（清晰度、具体性）
- source_reliability: 来源可靠性（直接陈述 > 推断）
- contextual_clarity: 上下文清晰度（是否有歧义）
```

## 矛盾严重性分级

### 分级标准

| 矛盾分数 | 置信度 | 严重级别 | 处理策略 |
|---------|-------|---------|---------|
| <0.3 | 任何 | 轻微 | 仅记录，不处理 |
| 0.3-0.5 | ≥0.7 | 轻度 | 观察，等待自然澄清 |
| 0.3-0.5 | <0.7 | 不确定 | 收集更多证据 |
| 0.5-0.7 | ≥0.8 | 中度 | 温和澄清机会 |
| 0.5-0.7 | <0.8 | 待验证 | 验证后再决定 |
| 0.7-0.9 | ≥0.9 | 严重 | 主动澄清询问 |
| >0.9 | ≥0.9 | 非常严重 | 立即关注，可能干预 |

### 分级考虑因素

#### 价值重要性权重
涉及核心价值的矛盾更严重：
- 健康、安全、家庭：权重 × 1.5
- 职业发展、财务：权重 × 1.3
- 兴趣爱好、日常偏好：权重 × 1.0

#### 后果严重性
矛盾可能导致的后果：
- 健康风险：严重性 +0.3
- 关系损害：严重性 +0.2
- 机会损失：严重性 +0.2
- 自我认同冲突：严重性 +0.1

## 矛盾处理策略

### 1. 静默记录策略
适用于轻微矛盾或低置信度情况。

#### 执行动作
- 记录矛盾细节到画像
- 降低相关字段置信度
- 标记为"需要进一步观察"
- 不主动提及或询问

#### 数据记录
```json
{
  "contradiction_record": {
    "type": "statement_vs_behavior",
    "field": "health_priority",
    "statement": "健康第一（置信度0.9）",
    "behavior": "连续3天熬夜（置信度0.85）",
    "score": 0.45,
    "confidence": 0.65,
    "action": "silent_record",
    "reason": "confidence_below_threshold",
    "monitoring_plan": "observe_7_days"
  }
}
```

### 2. 观察等待策略
适用于轻度矛盾，等待自然澄清机会。

#### 执行动作
- 记录并监控矛盾发展
- 等待相关话题自然出现
- 准备澄清问题但不主动发起
- 分析矛盾模式变化

#### 触发澄清条件
当以下条件满足时转为主动澄清：
1. 矛盾持续或加剧
2. 相关话题自然出现
3. 用户状态适合深入讨论
4. 有足够对话上下文

### 3. 主动澄清策略
适用于中度以上矛盾，需要理解原因。

#### 澄清问题设计原则
- **非评判性**：聚焦理解而非指责
- **开放性**：提供多种可能解释
- **支持性**：表明目的是更好支持用户
- **时机恰当**：选择用户状态好的时候

#### 澄清问题模板

**模板1：探索性询问**
```
"关于[话题]，我注意到一个可能需要澄清的点：
你之前提到[X]，但最近[Y]。
能帮我理解这是情况变化、不同情境，还是我理解有误？"
```

**模板2：多可能性提供**
```
"在[话题]上，我收集到一些不同信息：
1. [证据A]表明[X]
2. [证据B]表明[Y]

这可能是因为：
A. 情况最近有变化
B. 不同情境要求不同方式
C. 我的理解需要调整
D. 其他原因：[请说明]"
```

**模板3：价值冲突探索**
```
"看起来在[情境]中，[价值A]和[价值B]有些张力。
你通常如何平衡这两者？
还是在这种情况下有明确的优先级？"
```

### 4. 干预支持策略
适用于严重矛盾，涉及核心价值或风险。

#### 适用条件
- 涉及健康、安全风险
- 导致用户明显痛苦或困惑
- 用户主动请求帮助解决矛盾
- 矛盾严重影响决策能力

#### 支持方式
- 帮助理清价值优先级
- 提供决策框架
- 建议外部专业帮助（如需要）
- 支持制定一致性计划

## 矛盾学习与画像更新

### 矛盾解决后的更新

#### 用户明确澄清
当用户明确解释矛盾原因时：
```
if (用户提供合理解释) {
  更新相关字段值；
  调整置信度；
  记录解释作为新证据；
  标记矛盾为"已解决"；
}
```

#### 发现新模式
矛盾可能揭示新的用户模式：
- **情境特异性**：在某些情境下行为不同
- **价值演进**：价值优先级随时间变化
- **认知复杂性**：用户持有复杂、多层面的观点

### 画像置信度调整

#### 基于矛盾调整置信度
```
if (检测到矛盾) {
  相关字段置信度 = 原置信度 × (1 - 矛盾分数 × 严重性因子)；
  最低限制：置信度 ≥ 0.3；
}
```

#### 基于澄清恢复置信度
```
if (矛盾得到合理解释) {
  相关字段置信度 = min(原置信度 + 澄清质量 × 0.2, 1.0)；
}
```

## 伦理考虑

### 隐私保护
矛盾检测中的隐私保护：
- 不记录无争议的私人细节
- 矛盾数据加密存储
- 用户可查看和删除矛盾记录
- 矛盾数据不用于非直接相关分析

### 自主权尊重
- 矛盾澄清始终是邀请而非要求
- 用户可拒绝讨论任何矛盾
- 标记用户不愿讨论的话题
- 尊重用户自我认知的权利

### 避免标签化
- 矛盾是理解复杂性的机会，不是缺陷标签
- 聚焦具体情境而非人格评价
- 承认合理的情境性矛盾
- 避免过度心理学化解释
