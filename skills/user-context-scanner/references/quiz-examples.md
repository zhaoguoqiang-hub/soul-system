# Quiz交互示例库 - 验证与澄清策略

## Quiz设计原则

### 核心目标
1. **验证发现**：确认从记忆挖掘的模式是否准确
2. **澄清矛盾**：理解言行不一致的原因
3. **补充信息**：获取画像缺失的重要细节
4. **尊重自主**：以询问而非质问的方式

### 设计准则
- **开放性**：提供选择但允许自由回答
- **非评判性**：避免让用户感到被审查
- **时机恰当**：在相关话题自然出现时询问
- **频率适度**：避免Quiz疲劳

## Quiz类型分类

### 1. 模式验证Quiz
验证从行为模式中发现的潜在偏好或习惯。

#### 示例1：工作模式验证
```
🔍 发现新模式，想确认一下：

从你的工作记录看，你通常在上午处理复杂任务，
下午处理协调和沟通工作。

这是你的有意安排，还是自然形成的模式？

A. 是有意安排（上午效率高）
B. 自然形成，没特别注意
C. 实际情况不同（请描述）
```

#### 示例2：学习偏好验证
```
🔍 注意到一个模式：

当你学习新工具时，喜欢先看实际案例，
再研究理论文档。

这反映了你的学习偏好吗？

A. 是的，案例优先学习更有效
B. 不一定，看具体情况
C. 没意识到这个模式
```

#### 示例3：决策风格验证
```
🔍 从最近的决策记录发现：

你做重要决定时，通常会先收集数据，
再咨询1-2个信任的人，最后自己决定。

这准确描述你的决策过程吗？

A. 基本准确
B. 部分准确，但[补充细节]
C. 不准确，实际是[描述实际过程]
```

### 2. 矛盾澄清Quiz
当发现言行不一致时，理解背后的原因。

#### 示例1：健康承诺 vs 行为
```
🔍 发现一个可能需要澄清的点：

你多次强调"健康第一"，
但最近经常工作到深夜。

这是因为：

A. 近期项目紧急，临时情况
B. 优先级在实际执行时变化了
C. 健康定义不同（比如精神健康优先）
D. 其他原因：[请说明]
```

#### 示例2：兴趣表达 vs 实际投入
```
🔍 注意到一个不一致：

你说对"AI创业"很有兴趣，
但最近两周没讨论或研究相关话题。

这是因为：

A. 兴趣还在，但最近忙其他事
B. 兴趣方向有变化
C. 在研究但没在对话中提及
D. 实际情况：[请描述]
```

#### 示例3：沟通偏好 vs 实际行为
```
🔍 关于沟通方式有个疑问：

你提到喜欢"简洁直接"的沟通，
但最近经常问很多细节问题。

这是因为：

A. 不同情境需要不同沟通方式
B. 对重要事项需要更多细节
C. 我的理解可能有误
D. 其他解释：[请说明]
```

### 3. 信息补充Quiz
当画像信息不完整或模糊时，主动询问补充。

#### 示例1：职业发展目标
```
🔍 想更好地支持你的职业发展：

你提到关注"AI创业"和"自媒体"，
这两者在你未来的规划中是什么关系？

A. AI创业是核心，自媒体是宣传工具
B. 自媒体可能先做，积累后再AI创业
C. 两者平行发展，相互促进
D. 还在探索中，不确定关系
```

#### 示例2：家庭与工作平衡
```
🔍 关于工作生活平衡：

你有两个孩子，工作也很有追求。
在时间分配上，你更倾向于：

A. 明确划分：工作时间工作，家庭时间家庭
B. 灵活融合：根据需要随时切换
C. 以家庭为核心安排工作
D. 以工作为核心，家庭时间尽量保障
E. 实际情况更复杂：[请描述]
```

#### 示例3：压力应对方式
```
🔍 想了解你的压力应对偏好：

面对工作压力时，你通常更倾向于：

A. 自己静静思考解决方案
B. 找人讨论获取不同视角
C. 暂时放下，转移注意力
D. 立即行动，解决问题
E. 其他方式：[请描述]
```

### 4. 变化探测Quiz
检测用户偏好或情况是否发生变化。

#### 示例1：优先级变化
```
🔍 想确认一下优先级是否有变化：

三个月前，你提到"产品管理"是核心，
现在经常讨论"内容创作"。

这是：

A. 增加了新重点，但原有重点不变
B. 重点确实在转移
C. 只是临时兴趣，核心未变
D. 情况更复杂：[请说明]
```

#### 示例2：习惯变化
```
🔍 注意到作息模式有变化：

之前你通常在23:00前休息，
最近几周经常到00:00后。

这是因为：

A. 工作项目要求（临时性）
B. 作息习惯在改变
C. 其他原因影响睡眠
D. 数据可能不准确
```

## Quiz生成算法

### 触发条件

#### 1. 新模式发现
```
if (从记忆中发现新模式 && 置信度 < 0.7 && 无近期相关Quiz) {
  生成模式验证Quiz；
}
```

#### 2. 矛盾检测
```
if (检测到言行不一致 && 不一致程度 > 阈值 && 上次澄清 > 7天前) {
  生成矛盾澄清Quiz；
}
```

#### 3. 信息缺口
```
if (重要画像字段缺失 && 相关话题自然出现) {
  生成信息补充Quiz；
}
```

#### 4. 变化迹象
```
if (检测到行为模式变化 && 变化幅度 > 阈值) {
  生成变化探测Quiz；
}
```

### 时机选择策略

#### 最佳时机
- 相关话题自然出现在对话中
- 用户当前状态平静专注
- 非深夜或清晨打扰时段
- 近期无Quiz疲劳迹象

#### 避免时机
- 用户明显忙碌或压力大
- 深夜或清晨休息时段
- 连续对话已经很密集
- 近期Quiz接受率低

### 频率控制

#### 每日限制
- 最大Quiz次数：3次/天
- 同话题最小间隔：3天
- 全局最小间隔：2小时

#### 疲劳检测
监控用户对Quiz的回应：
- 接受并详细回答：保持频率
- 简短回答或忽略：降低频率
- 明确拒绝或不满：暂停3天

## Quiz回应处理

### 回应分类与处理

#### 1. 明确选择（A/B/C/D）
```
处理：
- 更新画像对应字段
- 根据选择调整置信度
- 记录用户明确偏好
```

#### 2. 自由描述
```
处理：
- 提取关键信息更新画像
- 可能触发进一步澄清问题
- 学习用户表达方式
```

#### 3. 模糊或回避
```
处理：
- 标记字段为"需要进一步验证"
- 降低相关推测置信度
- 等待更好时机或更多证据
```

#### 4. 纠正AI理解
```
处理：
- 立即更新画像纠正错误理解
- 感谢用户纠正
- 分析错误原因，优化检测算法
```

## 优化与学习

### Quiz效果评估

#### 短期指标
- 回应率（有回应Quiz占比）
- 回答质量（详细程度）
- 用户满意度（语气分析）

#### 长期指标
- 画像准确度提升
- 矛盾数量减少
- 用户信任度变化

### 个性化优化

#### 基于用户偏好调整
- **偏好详细回答的用户**：提供更多选项和解释
- **偏好简洁的用户**：简化问题，减少选项
- **思考型用户**：给予更长时间考虑
- **行动型用户**：提供直接明确的选择

#### 基于时机效果优化
分析不同时段的Quiz接受率：
- 找出用户最佳回应时段
- 避免低接受率时段
- 根据用户作息调整发送时间

### 模板迭代

#### A/B测试不同表述
对同一类Quiz测试不同表述：
- 直接 vs 委婉
- 详细 vs 简洁
- 正式 vs 随意
- 选择式 vs 开放式

选择效果最好的模板作为默认。

## 特殊情况处理

### 用户拒绝Quiz
当用户明确表示不想回答时：
1. 立即停止当前Quiz
2. 道歉如果造成打扰
3. 标记该话题为"用户不愿讨论"
4. 调整未来Quiz策略

### 敏感话题处理
涉及隐私或敏感话题时：
1. 提供退出选项
2. 明确说明目的和数据使用
3. 允许部分回答或不回答
4. 尊重用户边界

### 连续Quiz优化
当需要多个相关Quiz时：
1. 优先最重要的问题
2. 合并相关问题
3. 分散在不同对话中
4. 明确说明关联性
