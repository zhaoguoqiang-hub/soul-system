# 用户画像系统

让AI读懂你，然后记住你是谁。

## 核心价值

不是打标签，是**让USER.md活起来**。

USER.md是强哥的静态档案——但它应该是动态生长的。每次对话中发现的强哥的新特征，都应该同步到USER.md里，让AI每次读的时候都更精准。

```
用户透露信息 → 积累 → 画像更新 → USER.md同步 → AI更懂强哥
```

## 画像信息来源

### 直接信息（用户明确说的）
- "我是个产品经理" → 提取：职业
- "我一般周末不工作" → 提取：作息习惯
- "我老婆叫芳" → 提取：家庭成员
- "我有俩孩子" → 提取：家庭成员

### 间接信息（AI推断的）
- 连续3次在深夜对话 → 推断：可能是夜猫子
- 反复讨论某个话题 → 推断：当前关注点
- 语气变化规律 → 推断：情绪模式

## 画像字段

```
USER.md 中需要动态更新的字段：
- 职业/角色
- 作息习惯
- 家庭角色
- 当前关注领域
- 决策风格
- 沟通偏好
- 情绪模式
- 禁忌/底线
```

## 触发时机

用户透露以下类型的信息时，触发画像更新：

| 信息类型 | 识别模式 | 写入USER.md |
|---------|---------|------------|
| 职业 | "我是..."、"做...的" | 职业 |
| 作息 | "我一般..."、"习惯..." | 作息习惯 |
| 家庭 | "我老婆..."、"孩子..." | 家庭 |
| 偏好 | "我喜欢..."、"我不喜欢..." | 偏好 |
| 决策风格 | "我觉得...比...重要" | 决策风格 |
| 底线 | "我不能接受..."、"绝对不..." | 禁忌 |
| 时间节奏 | 连续出现的时间模式 | 作息 |

## 更新机制

### 即时更新（强烈信号）
用户**主动明确说**了什么，直接更新：
```
用户："我一般周末上午不工作"
→ 立即写入 USER.md: 作息习惯
```

### 积累后更新（弱信号）
单次信息不足以确定模式，需要积累：
```
第1次说"最近好累" → 标记：疲惫信号×1
第2次说"最近好累" → 标记：疲惫信号×2
第3次说"最近好累" → 确认：持续疲惫模式 → 写入画像
```

### 更新阈值
```
标签类信息：3次确认后写入
习惯类信息：2次确认后写入
底线/禁忌：1次确认即写入
```

## 调用 soul_context

```
工具：soul_context
动作：add 或 merge
参数：
  key: <字段名>
  value: <值>
```

## 写回 USER.md

当画像积累到一定量，启动"写回"流程：

1. 调用 soul_context(action="get") 获取当前画像
2. 分析是否有新的确定性信息
3. 将变化写入 USER.md 对应字段
4. 记录更新日志（方便回溯）

## 与 narrative-memory 的配合

narrative-memory 捕捉**叙事片段**（画面感）
user-context-scanner 提取**用户特征**（结构化）

两者关系：
- narrative-memory 告诉AI"强哥在那件事上是什么状态"
- user-context-scanner 告诉AI"强哥一般是什么样的人"

**两者结合 = 既有画面，又有理解**

## 重要提醒

- **用户主动说 > AI推断**：强哥亲口说的优先级最高
- **宁慢不准错**：标签一旦写错，后面很难修正
- **可回溯**：每次更新记录时间戳，方便知道"这个是什么时候加的"
- **服务理解，不是归档**：画像是为了让AI更懂强哥，不是为了收集数据
