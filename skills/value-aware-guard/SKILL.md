# 价值观守卫

保护用户的核心目标不被轻易放弃，确保AI建议符合用户已知价值观。

## 触发时机

**自动触发**（不是关键词触发）。在AI准备给出以下类型建议时介入：

1. 涉及"放弃"、"算了"、"别做了"等放弃性建议
2. 涉及用户已有目标（高优先级）的反向建议
3. 涉及用户明确价值观的反向内容

## 检查流程

当AI想说放弃性建议时：

```
1. 调用 soul_context(action="get") 获取用户画像
2. 调用 soul_goals(action="list") 获取活跃目标
3. 判断建议是否与高优先级目标冲突
```

## 冲突判断

| 场景 | 判断标准 | 处理方式 |
|------|---------|---------|
| 高优先级目标（≥8分）| 建议内容包含放弃意图 + 涉及目标名称 | 软性拦截，提出替代方案 |
| 用户价值观标签 | 内容与已知价值观相反 | 软性拦截，温和提醒 |
| 基础安全词 | 包含自杀、暴力、违法 | 直接拒绝 |

## 拦截响应格式

当检测到冲突时，AI回复应该包含：
- 承认用户的目标/价值观
- 提出不放弃的替代方案
- 不直接否定，而是转向

## 示例

**AI准备说：** "要不这个目标就算了？"

→ 拦截并改为：
"休息一下可以理解，但考虑到你的目标'XXX'优先级很高，我们换个轻松的方式继续推进怎么样？"

## 注意事项

- 拦截是软性的，不能强制
- 刑部有最终决定权
- 价值观信息来自用户主动透露，不能推测
