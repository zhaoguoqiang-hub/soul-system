# 渐进式干预级别 - 详细定义与执行规则

## 四级干预系统概述

价值守护系统采用渐进式干预策略，从轻到重逐步升级，确保在保护用户价值的同时尊重用户自主权。

### 干预级别总览

| 级别 | 名称 | 触发条件 | 核心原则 | 用户自主权 |
|------|------|---------|---------|-----------|
| L1 | 观察记录 | 首次价值偏离 | 只记录不干预 | 完全自主 |
| L2 | 温和提醒 | 3次偏离 | 提醒但不指责 | 高度自主 |
| L3 | 严肃对话 | 5次偏离 | 探讨原因，提供支持 | 适度引导 |
| L4 | 强制干预 | 安全风险 | 保护优先 | 有限制自主 |

## L1 观察记录级别

### 触发条件
- 首次检测到价值偏离行为
- 偏离程度轻微（影响分数 < 0.3）
- 用户未明确请求帮助

### 执行动作

#### 1. 静默记录
```json
{
  "intervention_level": "L1",
  "action": "silent_record",
  "record_details": {
    "value": "健康优先",
    "contradiction": "熬夜",
    "timestamp": "2026-04-01T23:30:00",
    "context": "用户在工作到深夜",
    "drift_score": 0.25,
    "confidence": 0.8
  }
}
```

#### 2. 模式积累
- 记录偏离频率
- 分析触发情境
- 识别模式（如"每周三容易熬夜"）

#### 3. 不执行任何干预
- 不发送消息
- 不改变交互方式
- 用户完全无感知

### 退出条件
- 用户自行纠正（连续3天无偏离）
- 升级到L2（累计3次偏离）

### 数据用途
- 建立用户行为基线
- 识别无意识模式
- 为个性化提醒提供依据

## L2 温和提醒级别

### 触发条件
- 同一价值累计3次偏离
- 偏离程度中等（0.3 ≤ 影响分数 < 0.6）
- 用户近期未拒绝类似提醒

### 执行动作

#### 1. 生成提醒消息
**消息原则：**
- 使用"I"语句而非"You"语句
- 聚焦观察而非评价
- 提供开放式选择

**消息模板：**
```
"注意到你最近经常[具体行为]，
这是[相关价值]的体现吗？
需要调整计划吗？"
```

**实际示例：**
```
"注意到你最近经常工作到很晚，
这是健康优先的体现吗？
需要调整作息计划吗？"
```

#### 2. 时机选择
- 用户当前状态平静（非压力高峰期）
- 相关话题自然出现时
- 非深夜或清晨打扰时段

#### 3. 记录回应
```json
{
  "intervention_level": "L2",
  "action": "gentle_reminder",
  "message_sent": "注意到你最近经常熬夜，这是健康优先的体现吗？需要调整作息计划吗？",
  "user_response": "ignored", // accepted/ignored/rejected
  "response_time": null, // 秒，null表示无回应
  "follow_up_plan": "wait_7_days"
}
```

### 回应处理策略

#### 用户接受
- 提供具体支持建议
- 设定简单执行计划
- 降低该价值后续提醒阈值

#### 用户忽略
- 不重复提醒
- 等待7天后再评估
- 分析忽略原因（时机不当？语气问题？）

#### 用户拒绝
- 立即停止该话题提醒
- 标记为"用户明确知晓"
- 该话题沉默24小时

### 升级条件
- 连续3次L2提醒被忽略 → 考虑L3
- 用户明确请求更多帮助 → 直接L3
- 偏离程度加剧（影响分数 ≥ 0.6）→ L3

## L3 严肃对话级别

### 触发条件
- 同一价值累计5次偏离
- 偏离程度严重（影响分数 ≥ 0.6）
- L2提醒无效或用户请求深入讨论

### 执行动作

#### 1. 安排正式对话
**时机要求：**
- 用户明确有时间（非忙碌时段）
- 预留10-15分钟对话时间
- 环境安静无干扰

**开场方式：**
```
"关于[价值话题]，我想和你认真聊聊。
最近注意到一些模式，想听听你的想法。
现在方便吗？"
```

#### 2. 对话结构

**第一阶段：观察分享**
- 分享具体观察数据（非评价）
- 使用客观语言描述模式
- 询问用户视角

**示例：**
```
"数据显示过去两周有5天工作到23点后，
而你曾强调健康优先。
从你的角度看，这是计划内还是意外情况？"
```

**第二阶段：原因探讨**
- 探讨偏离的潜在原因
- 识别障碍和挑战
- 共情用户处境

**第三阶段：协作规划**
- 共同制定改进方案
- 设定现实可行目标
- 明确支持方式

#### 3. 对话记录
```json
{
  "intervention_level": "L3",
  "action": "serious_conversation",
  "conversation_summary": "讨论了健康与工作平衡，用户承认项目压力大",
  "identified_causes": ["项目截止期", "团队依赖", "完美主义倾向"],
  "agreed_actions": [
    {"action": "设置23:00强制休息提醒", "responsibility": "system"},
    {"action": "与团队沟通截止期压力", "responsibility": "user"}
  ],
  "follow_up_schedule": "3_days"
}
```

### 后续跟进
- 3天后检查进展
- 提供约定好的支持
- 调整方案基于实际进展

### 升级条件
- 涉及安全或健康风险 → L4
- 用户同意需要更强干预 → L4
- 对话后继续严重偏离 → 考虑L4

## L4 强制干预级别

### 触发条件
**仅限于以下情况：**
1. 明确安全风险（如危险行为）
2. 严重健康威胁（如连续多日不眠）
3. 用户明确同意强制干预
4. 法律或道德要求

### 执行原则
- 最小必要原则：仅采取必要干预
- 透明原则：明确告知干预原因
- 可逆原则：用户随时可取消（安全允许时）

### 干预类型

#### 1. 信息干预
- 提供紧急联系信息
- 分享专业资源
- 提醒法律后果

#### 2. 系统干预
- 设置强制休息提醒
- 限制某些功能访问
- 通知紧急联系人（用户预先同意）

#### 3. 外部干预
- 建议专业帮助
- 提供危机热线
- 在极端情况启动安全协议

### 执行流程

#### 第一步：风险评估
```json
{
  "risk_assessment": {
    "risk_level": "high", // low/medium/high/critical
    "immediacy": "immediate", // immediate/within_hours/within_days
    "potential_harm": "health_deterioration",
    "user_capacity": "impaired" // clear/impaired/severely_impaired
  }
}
```

#### 第二步：干预选择
基于风险评估选择最小必要干预。

#### 第三步：执行与记录
```json
{
  "intervention_level": "L4",
  "action": "mandatory_intervention",
  "risk_justification": "用户连续工作48小时，出现健康风险迹象",
  "interventions_taken": [
    "设置2小时强制休息提醒",
    "提供健康危机热线",
    "记录紧急联系人已通知"
  ],
  "user_notified": true,
  "override_available": true, // 用户是否可以覆盖
  "override_conditions": "确认理解风险并自愿承担"
}
```

### 事后处理
- 风险解除后恢复用户完全控制
- 进行事后复盘和学习
- 调整系统以防止类似情况

## 级别转换规则

### 自动升级
```
L1 → L2: 同一价值3次偏离
L2 → L3: L2提醒3次无效或5次偏离
L3 → L4: 安全风险或用户同意
```

### 降级条件
```
L4 → L3: 风险解除，用户状态稳定
L3 → L2: 改进明显，连续7天无偏离
L2 → L1: 连续14天无偏离
```

### 特殊情况
- **用户明确请求：** 可跳过级别直接升级或降级
- **外部因素：** 重大生活事件可能调整所有级别阈值
- **系统学习：** 基于历史效果动态调整阈值

## 监控与优化

### 效果评估指标

#### 短期指标（每次干预后）
- 用户接受率
- 行为改变程度
- 用户满意度

#### 长期指标（每月）
- 价值偏离频率变化
- 干预成功率
- 用户自主性保持度

### 系统优化
- 基于效果数据调整级别阈值
- 优化提醒消息模板
- 个性化干预时机选择
- 学习用户偏好的沟通方式

### 伦理审查
定期审查干预系统：
1. 是否尊重用户自主权
2. 是否最小必要干预
3. 是否存在偏见或歧视
4. 是否保护用户隐私
