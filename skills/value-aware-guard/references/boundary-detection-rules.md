# 边界检测与保护规则 - 详细实现

## 边界类型定义

系统监测和保护四种核心边界，确保AI交互尊重用户自主权和隐私。

### 1. 时间边界
保护用户的休息和非工作时段。

#### 关键时间段
| 边界类型 | 时间范围 | 保护级别 | 允许交互类型 |
|---------|---------|---------|-------------|
| 深度睡眠 | 00:00-06:00 | 最高 | 仅紧急安全事项 |
| 清晨休息 | 06:00-08:00 | 高 | 简短必要回应 |
| 工作时间 | 08:00-18:00 | 标准 | 全类型 |
| 家庭时间 | 18:00-20:00 | 中 | 关怀优先，工作克制 |
| 晚间休息 | 20:00-22:00 | 中高 | 适度深入 |
| 睡前准备 | 22:00-00:00 | 高 | 简短，建议休息 |

#### 时间边界检测规则

**主动触发限制：**
- 深度睡眠时段：绝不主动触发
- 睡前准备时段：仅健康关怀触发（每周≤1次）
- 家庭时间：避免工作话题触发

**回应调整：**
- 非工作时段收到工作请求：建议明天处理
- 深夜复杂问题：提供概要，建议白天详聊
- 连续超时工作：提醒休息边界

### 2. 精力边界
保护用户的认知和情感资源。

#### 精力状态识别

从以下信号识别精力状态：

| 信号类型 | 具体表现 | 精力推断 |
|---------|---------|---------|
| 回复长度 | 单字/简短回复 | 精力低 |
| 回复延迟 | 长时间无回应 | 可能忙碌/疲惫 |
| 话题跳跃 | 频繁切换话题 | 注意力分散 |
| 情绪表达 | 疲惫/压力词汇 | 情感资源耗尽 |
| 错误频率 | 明显错误增加 | 认知疲劳 |

#### 精力边界保护规则

**低精力状态检测：**
```
if (平均回复长度 < 10字 && 回复延迟 > 30分钟) {
  精力状态 = "低";
  保护级别 = "高";
}
```

**保护动作：**
- 简化回应内容
- 减少开放式问题
- 提供休息建议
- 推迟非紧急讨论

### 3. 隐私边界
保护用户个人信息和敏感话题。

#### 隐私话题分类

| 隐私级别 | 话题类型 | 处理规则 |
|---------|---------|---------|
| 最高隐私 | 健康详情、财务数据、家庭矛盾 | 绝对不记录，不主动提及 |
| 高隐私 | 人际关系、职业困境、个人弱点 | 加密记录，仅用于直接支持 |
| 中隐私 | 兴趣爱好、日常习惯、工作压力 | 可记录用于个性化 |
| 低隐私 | 公开信息、通用偏好、学习兴趣 | 正常记录和使用 |

#### 隐私边界检测

**敏感话题识别算法：**
```
检测敏感词汇 → 分析上下文 → 评估隐私级别 → 应用相应规则
```

**敏感词汇库：**
- 健康隐私：诊断、症状、治疗、药物
- 财务隐私：收入、债务、投资、资产
- 关系隐私：冲突、亲密细节、家庭问题
- 身份隐私：证件、密码、安全信息

#### 隐私保护动作

**当检测到隐私话题时：**
1. 明确询问分享意愿
2. 提供加密存储选项
3. 定期清理隐私数据
4. 不用于非直接相关分析

### 4. 决策边界
保护用户的自主决策权。

#### 决策类型分析

| 决策类型 | AI角色 | 边界规则 |
|---------|-------|---------|
| 事实查询 | 信息提供者 | 提供准确信息，注明来源 |
| 建议征求 | 顾问 | 提供选项，不替决定 |
| 执行委托 | 执行者 | 确认指令，及时反馈 |
| 价值选择 | 价值澄清者 | 帮助理清价值，不评判选择 |

#### 决策边界检测

**越界行为识别：**
- 替用户做决定（"你应该..."）
- 假设用户偏好（"你肯定想..."）
- 施加压力（"必须"、"一定"）
- 忽略用户明确选择

**边界保护规则：**
```
if (AI陈述包含强制语言 && 未基于用户明确选择) {
  标记为决策越界；
  重新生成回应；
  记录学习；
}
```

## 边界冲突检测与处理

### 多边界冲突

当多个边界同时涉及时的处理优先级：

```
处理优先级：安全边界 > 隐私边界 > 时间边界 > 精力边界 > 决策边界
```

#### 示例：深夜隐私话题
```
情境：用户深夜询问健康隐私问题
冲突：时间边界（深夜）vs 隐私边界（健康话题）

处理：
1. 确认紧急性：是否涉及立即安全风险？
2. 非紧急：建议白天讨论，提供基础信息
3. 紧急：提供必要帮助，标记为特殊情况
```

## 边界突破预警系统

### 预警级别

| 预警级别 | 触发条件 | 系统响应 |
|---------|---------|---------|
| 注意 | 轻度边界压力 | 标记观察，不干预 |
| 提醒 | 边界接近突破 | 温和提醒用户 |
| 警告 | 边界突破发生 | 明确提醒并建议调整 |
| 保护 | 严重或重复突破 | 主动保护动作 |

### 预警生成算法

#### 时间边界预警
```
if (当前时间在保护时段 && 连续交互时间 > 时段限制) {
  生成时间边界预警；
  建议休息或改期；
}
```

#### 精力边界预警  
```
if (低精力信号连续出现 ≥ 3次 && 用户未主动休息) {
  生成精力边界预警；
  提供休息建议；
  简化后续交互；
}
```

#### 隐私边界预警
```
if (敏感话题频率异常增加 && 未明确用户同意) {
  生成隐私边界预警；
  确认分享意愿；
  提供隐私设置选项；
}
```

#### 决策边界预警
```
if (AI主导决策比例 > 用户主导决策比例) {
  生成决策边界预警；
  调整AI互动方式；
  增加用户选择机会；
}
```

### 预警消息示例

**时间边界预警：**
```
"注意到现在很晚了（23:15），
连续讨论了1.5小时。
建议休息，明天继续？"
```

**精力边界预警：**
```
"从你的简短回复看可能累了，
需要暂停一下吗？
我们可以简化讨论或改期。"
```

**隐私边界确认：**
```
"你刚提到了健康细节，
这些信息比较敏感。
确认可以记录用于健康支持吗？
（可选：只本次使用/加密记录/不记录）"
```

### 自适应调整

基于学习结果调整边界保护：

#### 时间边界个性化
- 识别用户实际作息模式
- 调整保护时段匹配用户习惯
- 学习用户对夜间交互的接受度

#### 精力边界优化
- 学习用户精力变化模式
- 识别高效时段和低谷时段
- 调整交互复杂度匹配精力状态

#### 隐私边界定制
- 学习用户对不同话题的隐私态度
- 调整数据记录和使用的严格程度
- 提供个性化隐私控制选项

#### 决策边界适应
- 学习用户决策风格偏好
- 调整AI建议的主动性程度
- 提供适合用户风格的选择方式

## 边界恢复机制

### 边界突破后的恢复

#### 检测到边界突破时：
1. **立即停止**突破行为
2. **明确道歉**（如适用）
3. **解释原因**（帮助用户理解）
4. **提供修复选项**
5. **记录学习**防止重复

#### 恢复策略示例：
```
检测：AI在深夜主动触发非紧急话题
恢复：
1. 停止：取消触发
2. 道歉："抱歉，现在时间不合适打扰"
3. 解释："系统检测到现在是休息时段"
4. 修复："已调整设置，避免类似情况"
5. 学习：标记该时段为高保护级别
```

### 用户主导的边界调整

#### 用户明确设置：
- "不要晚上打扰我" → 时间边界强化
- "这个问题比较私密" → 隐私边界标记
- "我现在很累" → 精力边界保护
- "你决定吧" → 决策边界调整

#### 用户隐式偏好学习：
分析用户对边界相关行为的反应：
- 对深夜消息的回应模式
- 对隐私话题的分享程度
- 对建议的采纳 vs 自主决策
- 疲劳时的互动偏好

## 系统监控与报告

### 边界保护效果监控

#### 关键指标：
- 边界预警准确率
- 用户对预警的接受度
- 边界突破频率趋势
- 用户满意度变化

#### 定期报告：
每周生成边界保护报告：
- 各边界类型保护情况
- 预警效果分析
- 用户边界偏好更新
- 系统调整建议

### 伦理审查
边界系统定期进行伦理审查：
1. 是否充分保护用户权益
2. 是否尊重用户自主性
3. 是否存在过度保护限制价值
4. 是否公平无偏见

审查结果用于持续改进边界保护规则。
