# 主动触发系统

让AI从"被动响应"变成"懂你的陪伴"。

## 核心区别

旧思路：目标3天没更新 → 提醒
新思路：知道强哥的生活节奏 → 在对的时机说对的话

## 触发来源（5种）

| 类型 | 说明 | 触发条件 |
|---------|------|---------|
| goal_based | 目标进展提醒 | 子目标完成 / 里程碑 |
| time_habit | 时间习惯触发 | 用户平时在这个时间点做的事 |
| context | 上下文变化 | 检测到外部事件影响用户目标 |
| reflection | 反思洞察 | 后台反思产生了有价值的发现 |
| pending_task | 未完成任务 | 有任务超过4小时未闭环 |

## 触发前必检

```
1. soul_focus(action="peek") → 专注模式开启，不触发
2. 当前时间在允许范围内 → 深夜不触发
3. soul_proactive(action="status") → 今日次数已用完，不触发
4. 检查冷却时间 → 两次触发至少间隔4小时
```

## 学习机制

- 拒绝3次 → 该类型静默24小时
- 接受 → 权重+10%
- 专注模式 → 积累建议，不发送

## 冷却机制

- 每天最多3次
- 每次间隔至少4小时
- 同一话题1小时不触发2次
- 拒绝3次，静默24小时
