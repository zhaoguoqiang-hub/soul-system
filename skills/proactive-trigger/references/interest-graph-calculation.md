# 话题热度衰减模型 - 详细计算规则

## 话题状态定义

每个话题在系统中可以处于三种状态之一：

### 🔥 活跃状态 (Active)

**触发能力：** 可主动触发  
**定义条件：** 最近24小时内被提及 ≥ 3次  
**维持条件：** 每3天至少被提及1次，否则降级为浮现状态  

### 🌱 浮现状态 (Emerging)  

**触发能力：** 不可主动触发（仅作为每日简报素材）  
**定义条件：** 最近24小时内被提及 1-2次  
**升级条件：** 累计提及 ≥ 3次（可在不同对话中） → 升级为活跃  
**降级条件：** 7天内未被提及 → 降级为休眠  

### ❄️ 休眠状态 (Dormant)

**触发能力：** 不可主动触发  
**定义条件：** 超过7天未被提及  
**激活条件：** 用户主动提及任意次数 → 立即激活为浮现状态  

## 状态转换规则

### 自动转换（基于时间衰减）

```
活跃 → 浮现：3天内未被提及
浮现 → 休眠：7天内未被提及
```

### 用户驱动转换

```
休眠 → 浮现：用户主动提及任意次数（立即激活）
浮现 → 活跃：累计提及次数 ≥ 3次（可跨多个对话）
```

## 提及次数计算规则

### 有效提及判定

1. **直接提及**：用户在对话中明确说出话题关键词
2. **相关提及**：用户讨论与话题高度相关的内容（需>80%语义相关）
3. **主动询问**：用户询问关于话题的信息
4. **被动回应**：用户回应AI关于该话题的询问

### 不计入的情况

1. AI单方面提及（用户未回应）
2. 偶然提到（上下文不相关）
3. 否定性提及（"我不关心这个话题"）

## 热度评分计算

```
TopicHeat = (recent_mentions / max_mentions) × time_decay_factor

其中：
- recent_mentions = 最近24小时内的提及次数（上限10次）
- max_mentions = 10（归一化系数）
- time_decay_factor = e^(-hours_since_last/48)

热度评分范围：0.0 - 1.0
```

### 热度阈值

| 热度评分 | 对应状态 | 触发建议 |
|---------|---------|---------|
| 0.7-1.0 | 🔥 活跃 | 可主动触发（每天最多1次） |
| 0.3-0.7 | 🌱 浮现 | 可作为简报素材 |
| 0.0-0.3 | ❄️ 休眠 | 不主动提及 |

## 话题发现与注册

### 自动发现规则

1. **关键词提取**：从用户对话中提取名词短语和专有名词
2. **语义聚类**：相似话题自动合并（如"AI应用"和"人工智能"）
3. **重要性筛选**：只注册提及次数≥2的话题

### 手动管理

用户可通过以下方式管理话题：
- 明确说"我对X话题感兴趣" → 强制注册为浮现状态
- 明确说"不要跟我提X" → 标记为禁止话题
- 长期忽略某个话题 → 自动进入休眠

## 冷却期管理

### 触发冷却

```
活跃话题触发后进入冷却期：
- 同一话题：冷却1小时
- 同类型话题：冷却30分钟
```

### 降级冷却

```
状态降级后：
- 活跃→浮现：3天内不自动升级
- 浮现→休眠：7天内不自动激活
```

## 与其他系统的集成

### 与user-context-scanner集成

user-context-scanner发现的新兴趣领域会自动注册为浮现状态话题。

### 与value-aware-guard集成

如果某个话题与用户核心价值冲突（如"熬夜工作" vs "健康优先"），会降低其热度评分。

### 与mood-simulator集成

根据用户当前情绪状态调整话题触发权重：
- 积极活跃状态：所有话题热度 × 1.2
- 疲惫状态：所有话题热度 × 0.7
