# 沉默等待指数 - 详细计算公式

## 核心概念

沉默等待指数（SilenceIndex）衡量用户不互动的时间长度，用于调整主动触发的倾向性。

**基本原则：** 沉默时间越长，主动触发的可能性越高，但需避免打扰。

## 基础沉默指数计算

### 时间分段与基础值

| 沉默时长 | 基础指数 | 说明 |
|---------|---------|------|
| 0-30分钟 | 0.1 | 刚聊完，极克制 |
| 30分钟-2小时 | 0.2 | 正常间隔 |
| 2-6小时 | 0.5 | 适度等待 |
| 6-12小时 | 0.8 | 适当主动 |
| 12-24小时 | 1.0 | 强主动 |
| 24+小时 | 1.2 | 超强主动（但有上限） |

### 计算公式

```
SilenceIndex = base_value × activity_factor × time_of_day_factor

其中：
- base_value: 根据上表的基础值
- activity_factor: 用户活动模式因子（0.8-1.2）
- time_of_day_factor: 时间段调整因子（0.7-1.3）
```

## 用户活动模式因子

基于用户历史互动模式调整：

### 高频互动用户
- 特征：平均每天互动 ≥ 5次
- activity_factor = 0.8
- 说明：高频用户需要更克制

### 中频互动用户  
- 特征：平均每天互动 2-4次
- activity_factor = 1.0
- 说明：标准调整

### 低频互动用户
- 特征：平均每天互动 ≤ 1次
- activity_factor = 1.2
- 说明：低频用户可适当更主动

## 时间段调整因子

考虑用户的作息习惯：

| 时间段 | 调整因子 | 说明 |
|-------|---------|------|
| 深夜（00:00-06:00） | 0.3 | 绝对不打扰 |
| 清晨（06:00-08:00） | 0.7 | 轻度克制 |
| 上午（08:00-12:00） | 1.0 | 标准 |
| 午休（12:00-14:00） | 0.8 | 轻度克制 |
| 下午（14:00-18:00） | 1.0 | 标准 |
| 傍晚（18:00-20:00） | 0.9 | 轻度克制 |
| 晚间（20:00-22:00） | 1.1 | 稍积极 |
| 睡前（22:00-00:00） | 0.5 | 重度克制 |

## 综合触发评分计算

```
TriggerScore = SilenceIndex × TopicHeat × Urgency × ContextRelevance

其中：
- SilenceIndex: 沉默等待指数（0.0-1.2）
- TopicHeat: 话题热度评分（0.0-1.0）
- Urgency: 紧迫度权重（见下表）
- ContextRelevance: 上下文相关度（0.5-1.0）
```

### 紧迫度权重表

| 紧迫类型 | 权重 | 触发条件 |
|---------|------|---------|
| 安全风险 | 2.0 | 涉及安全、健康风险 |
| 目标deadline | 1.5 | 目标截止时间 < 24小时 |
| 健康提醒 | 1.3 | 健康相关，用户有承诺 |
| 机会提醒 | 1.0 | 发现相关机会 |
| 里程碑庆祝 | 0.9 | 达成重要里程碑 |
| 日常关怀 | 0.8 | 常规检查 |
| 信息分享 | 0.7 | 相关信息更新 |

### 触发阈值

| TriggerScore | 触发决策 | 说明 |
|-------------|---------|------|
| ≥ 1.5 | 立即触发 | 高价值高紧迫 |
| 1.0-1.5 | 建议触发 | 有价值适度紧迫 |
| 0.7-1.0 | 谨慎触发 | 低价值或低紧迫 |
| < 0.7 | 不触发 | 价值不足 |

## 上下文相关度计算

### 相关度因子

```
ContextRelevance = max(topic_relevance, recent_context_relevance, historical_pattern_relevance)

其中：
- topic_relevance: 与当前对话话题的相关度（0-1）
- recent_context_relevance: 与最近3条消息的相关度（0-1）
- historical_pattern_relevance: 与历史模式的相关度（0-1）
```

### 相关度判定规则

1. **直接相关**：同一话题 → 相关度 = 1.0
2. **领域相关**：同一领域（如"工作"） → 相关度 = 0.8
3. **间接相关**：有逻辑关联 → 相关度 = 0.6
4. **不相关**：无关联 → 相关度 = 0.3（最小值）

## 动态调整机制

### 基于反馈的调整

```
用户接受触发 → SilenceIndex阈值降低0.05（更容易触发）
用户忽略触发 → 无变化
用户拒绝触发 → SilenceIndex阈值提高0.1（更克制）
连续3次拒绝 → 该话题类型沉默24小时
```

### 基于时间模式的调整

分析用户最佳互动时间：
- 识别用户响应率高的时间段
- 在这些时间段适当提高SilenceIndex
- 在用户通常不互动的时间段降低SilenceIndex

## 特殊情况处理

### 用户明确忙碌

检测信号：用户说"在忙"、"稍后"、"现在没空"
处理：SilenceIndex强制设置为0.1，持续2小时

### 用户情绪低落

从mood-simulator接收"情绪波动"或"中度疲惫"信号
处理：Urgency权重 × 0.7，降低触发倾向

### 重要日子

检测到生日、纪念日等特殊日期
处理：关怀类触发Urgency权重 × 1.5
