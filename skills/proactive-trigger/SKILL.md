# 主动触发系统

让AI从被动响应变为主动关怀。

## 触发条件

以下任一条件满足时，可以主动发起：

### 条件A：目标停滞
- 某个目标超过3天无进展
- 且进度低于50%
→ 发送关怀性提醒（不是催促）

### 条件B：目标即将完成
- 某个目标进度在80%-99%之间
→ 发送鼓励性提醒，趁热打铁

### 条件C：状态捕捉
- 用户表达了疲惫、压力等负面情绪
- 但没有要求帮助
→ 适度关心，不强行介入

## 主动前检查

每次主动触发前，必须检查：

```
1. soul_focus(action="peek") → 如果专注模式开启，不触发
2. soul_proactive(action="status") → 剩余次数不足，不触发
3. 当前时间是否在7:00-22:00之间 → 深夜不触发
```

## 触发时间窗口

| 时段 | 可触发 | 说明 |
|------|--------|------|
| 07:00-12:00 | ✅ | 上午黄金时间 |
| 12:00-14:00 | ⚠️ | 午休，谨慎触发 |
| 14:00-18:00 | ✅ | 下午工作时间 |
| 18:00-22:00 | ✅ | 晚间，适度触发 |
| 22:00-07:00 | ❌ | 深夜不打扰 |

## 消息格式

主动触发的消息必须：
1. **有具体理由**："看到你'XXX'目标3天没更新了"
2. **非侵入式**：用问句结尾，不是命令
3. **提供帮助**：暗示可以协助拆解任务

## 示例

**目标停滞：**
"看到你'产品设计'目标已经3天没更新了，是不是遇到了什么困难？需要我帮你把任务拆小一点吗？"

**目标即将完成：**
"太棒了！'第一版开发'已经到90%了！今天要不要冲刺一下把它搞定？"

**状态关怀：**
"注意到你说最近比较累，要注意休息啊。需要我帮忙整理一下待办事项吗？"

## 冷却机制

每天最多 **3次** 主动触发。
两次触发间隔至少 **4小时**。

## 注意事项

- 宁可不发，不要乱发
- 用户在忙碌时不触发（最近5分钟有交互=忙碌）
- 专注模式用户完全跳过
